Task 1: Creating a Taxi class

[10 marks]

We will need to somehow represent a Taxi in the program. Create a class called Taxi to hold the following
information:
 taxiId, an integer
 numberPlate, a String
 mileage, an integer
 electricVehicle, a boolean
Your Taxi class must have the above instance variables with the exact names and types provided.
The taxiId instance variable is an internal (to the object) ID, which starts from 499101 for the rst Taxi
object created by the program, and increases by 1 for every subsequent Taxi object. All other instance
variables are required for a Taxi object to exist. In other words, it must be impossible to create a Taxi
object without supplying values for each of the other instance variables.
The class must provide both an accessor and mutator for each of its instance variables, except the taxiId
which is initialised automatically, and must only have an accessor.
The class must also provide a toString() method which should return a string that matches the below
format exactly:
{taxiId; numberPlate; mileage; electricVehicle}
For example, for a Taxi where taxiId=499101, numberPlate=AA00BBB, mileage=1234
and electricVehicle=true, the output must be:
{499101; AA00BBB; 1234; true}

Task 3: Expanding the Taxi class

[10 marks]

Every taxi has an assigned driver associated with it. Create a Driver class to hold the following information:
 fullName, a String
 licenseCode, a String
A Driver object should not be creatable without values to both attributes being supplied. Moreover,
both should have an accessor and a mutator. Like with all classes you write for this assignment, you
must ensure this class follows the principles of encapsulation.
Besides a driver, each taxi must also have a type. The company has three types of taxi: SMALL, REGULAR
and MINIBUS. To represent this, you should create an enum called TaxiType with these three values.
In your Taxi class, create an attribute to hold the Driver of that taxi and another to hold the type of
taxi it is. It should not be possible for an instance of the Taxi class to be created without a type supplied,
however it should be possible for a Taxi to be created with, and without a Driver supplied. Think of
the implications this requirement has for constructors.
Both of these instance variables should have an accessor, but only the driver instance variable should
have a mutator. Do not modify toString.

Task 4: Adding some methods...

[15 marks]

Time to start adding functionality to our classes.
The license identier of a Driver is a string, which will always be exactly 10 characters long. The last 3
characters of that string, determine what type of taxi a driver can drive, as follows:
 If the 8th character from the left is `M', the driver can drive a MINIBUS
 If the 9th character from the left is `R', the driver can drive a REGULAR taxi
 If the 10th character from the left is `S', the driver can drive a SMALL taxi

Note: A driver may be able to drive multiple dierent types of taxi.
Write a method in the Driver class called canDrive that takes a TaxiType parameter and returns true
if the driver can drive it, or false otherwise.
Besides knowing each driver's ability to drive a taxi type, we want to have a way of calculating the cost
of a ride. Create a method in your Taxi class called calculateCost that takes in an amount of miles,
and returns the cost of driving the taxi this length. For some distance M the cost is calculated as follows:
 If the taxi is of type SMALL: cost = M × 0.93
 If the taxi is of type REGULAR: cost = M × 1.21
 If the taxi is of type MINIBUS: cost = M × 2.55
A 30% discount is applied to the calculated cost, if the taxi is electric.
Think of how the TaxiType enum may support the implementation of this feature.
